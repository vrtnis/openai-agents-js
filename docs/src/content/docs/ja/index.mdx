## 概要

[OpenAI Agents SDK for TypeScript](https://github.com/openai/openai-agents-js) は、最小限の抽象化で軽量かつ使いやすいパッケージとして、エージェント型の AI アプリを構築できるようにします。これは以前のエージェント向け実験プロジェクトである [Swarm](https://github.com/openai/swarm/tree/main) の本番環境向けアップグレード版で、[Python 版](https://github.com/openai/openai-agents-python) も提供されています。Agents SDK にはごく少数の基本コンポーネントがあります。

- **エージェント** — instructions と tools を備えた LLM
- **ハンドオフ** — 特定のタスクを別のエージェントに委譲する仕組み
- **ガードレール** — エージェントへの入力を検証する仕組み

TypeScript と組み合わせることで、これらのコンポーネントはツールとエージェント間の複雑な関係を表現でき、学習コストを抑えつつ実運用レベルのアプリケーションを構築できます。さらに SDK には **トレーシング** が組み込まれており、エージェントフローの可視化とデバッグに加えて、評価やファインチューニングにも活用できます。

## Agents SDK を使う理由

SDK の設計原則は 2 つです。

1. 使う価値があるだけの機能を備えつつ、学習が容易なようにコンポーネントを最小限にする
2. すぐに使える状態で提供しつつ、挙動を細かくカスタマイズできる

主な特徴は次のとおりです。

- **Agent loop**: ツール呼び出し、結果の LLM への送信、LLM が完了するまでのループを自動で実行
- **TypeScript ファースト**: 新しい抽象概念を覚えることなく、言語機能だけでエージェントを連携・オーケストレーション
- **ハンドオフ**: 複数エージェント間での調整や委譲を行う強力な仕組み
- **ガードレール**: エージェント実行と並行して入力の検証を行い、失敗時には早期に停止
- **Function tools**: どんな TypeScript 関数でも自動スキーマ生成と Zod 検証付きツールに変換
- **トレーシング**: ワークフローの可視化・デバッグ・監視に加え、OpenAI の評価、ファインチューニング、蒸留ツールを活用可能
- **Realtime Agents**: 自動割り込み検知、コンテキスト管理、ガードレールなどを備えた高機能な音声エージェントを構築

```bash

```
